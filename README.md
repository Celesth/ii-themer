# ii-theme — illogical-impulse VSCode/VSCodium Theme

Syncs your editor color theme with [end-4's dots-hyprland](https://github.com/end-4/dots-hyprland) (illogical-impulse) Material You color system.

On startup (and optionally on wallpaper change), the extension reads your generated `colors.json` and builds a matching VSCode theme from the Material Design 3 palette.

## Requirements

- end-4 dots-hyprland (ii-qs branch / Quickshell version) installed and a wallpaper set
- `~/.local/state/quickshell/user/generated/colors.json` must exist

## How it works

1. Reads `colors.json` generated by matugen/kde-material-you-colors
2. Derives all editor, UI, and syntax token colors from the Material You palette (dark scheme)
3. Writes a generated theme JSON to extension storage and activates it
4. Optionally watches `colors.json` for changes (e.g. after `Super+Ctrl+T` wallpaper switch) and auto-reloads

## Commands

| Command | Description |
|---|---|
| `ii-theme: Reload from colors.json` | Manually re-apply the theme |

## Settings

| Setting | Default | Description |
|---|---|---|
| `ii-theme.colorsPath` | `""` | Override path to `colors.json` |
| `ii-theme.watchFile` | `true` | Auto-apply when wallpaper changes |

## Installation (manual / sideload)

```bash
# Clone or copy the ii-theme folder somewhere
cd ii-theme
npm install

# Package (requires vsce)
npm install -g @vscode/vsce
vsce package

# Install the .vsix
codium --install-extension ii-theme-1.0.0.vsix
# or for VSCode:
code --install-extension ii-theme-1.0.0.vsix
```

## VSCodium path difference

If VSCodium's extension storage is in a different path than expected, the extension still works — it uses the VS Code API's `context.globalStorageUri` for the generated theme file, which resolves correctly for both.

## Notes

- The extension rewrites `package.json`'s `contributes.themes` on each apply to point at the freshly generated theme file. This is necessary because VSCode requires theme contributions to be declared statically.
- A window reload is triggered when the theme is first applied. Subsequent wallpaper-change reloads also trigger a window reload to pick up token color changes.
- If you use the proprietary VSCode build, the paths match what end-4's own `material-code-set-color.sh` script targets. For VSCodium or other forks, set `ii-theme.colorsPath` if needed.
